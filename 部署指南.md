# 部署指南：GitHub Pages + 阿里云域名（无需备案）

> GitHub Pages 服务器在境外，域名解析到 GitHub 一般**不需要 ICP 备案**，适合先试用。

---

## 第一步：推送代码到 GitHub

### 1. 在 GitHub 新建仓库

- 打开 https://github.com/new
- 仓库名例如：`retail-ai-tools`（随意，建议英文）
- 选 **Public**
- **不要**勾选 README（本地已有代码）

### 2. 本地推送

在项目目录执行（把 `你的用户名` 和 `仓库名` 换成你的）：

```bash
cd "/Users/zhaochanghao/Desktop/pdf 图片转html"

git init
git add .
git commit -m "init: 新零售AI工具导航站"
git branch -M main
git remote add origin https://github.com/你的用户名/仓库名.git
git push -u origin main
```

---

## 第二步：开启 GitHub Pages

1. 打开仓库 → **Settings** → **Pages**
2. **Build and deployment** → Source 选 **GitHub Actions**
3. 回到 **Actions** 标签，确认 `Deploy to GitHub Pages` 工作流已成功（绿色 ✓）

部署成功后，临时访问地址为：

```
https://你的用户名.github.io/仓库名/
```

此时 `VITE_BASE_PATH` 已设为 `/`，若只用上述地址访问可能资源路径不对。**绑定自定义域名后**或改用用户/组织站点（见文末）即可正常访问。

---

## 第三步：绑定自定义域名

### 推荐：使用子域名（最简单）

例如：`tools.你的域名.com`

1. 仓库 **Settings → Pages → Custom domain**
2. 填入：`tools.你的域名.com`
3. 勾选 **Enforce HTTPS**（DNS 生效后会出现，稍等几小时）

### 也可用根域名

例如：`你的域名.com`（需配置 A 记录，见下一步）

---

## 第四步：阿里云 DNS 解析

登录 [阿里云域名控制台](https://dc.console.aliyun.com/) → 你的域名 → **解析设置** → **添加记录**

### 方式 A：子域名（推荐）

| 配置项 | 填写内容 |
|--------|----------|
| 记录类型 | CNAME |
| 主机记录 | `tools`（即 tools.你的域名.com） |
| 记录值 | `你的GitHub用户名.github.io` |
| TTL | 10 分钟 |

### 方式 B：根域名（你的域名.com）

添加 **4 条 A 记录**，主机记录均为 `@`，记录值分别为：

```
185.199.108.153
185.199.109.153
185.199.110.153
185.199.111.153
```

> 根域名无法在 GitHub 同时稳定使用 CNAME，因此用 A 记录。

---

## 第五步：等待生效

| 步骤 | 预计时间 |
|------|----------|
| DNS 解析生效 | 10 分钟 ~ 2 小时 |
| GitHub HTTPS 证书 | DNS 生效后最多 24 小时 |
| 首次访问 | 建议用无痕窗口测试 |

访问：`https://tools.你的域名.com`（换成你的域名）

---

## 后续更新网站

改完 `products.json`、封面图或代码后：

```bash
git add .
git commit -m "update: 更新产品信息"
git push
```

Actions 会自动重新构建部署，**域名不用改**。

## 常见问题

### 1. 页面空白 / 样式丢失

- 确认 `deploy.yml` 里 `VITE_BASE_PATH: /`
- 确认 Custom domain 已正确填写
- 强制刷新：`Cmd + Shift + R`

### 2. DNS 不生效

- 阿里云解析列表里确认记录状态为「正常」
- 用 https://dnschecker.org 查询 CNAME 是否全球生效

### 3. HTTPS 证书报错

- 等 DNS 完全生效后，GitHub 会自动申请证书
- Settings → Pages 里确认 **Enforce HTTPS** 已勾选

### 4. 想用 github.io/仓库名 访问（不绑域名）

把 `deploy.yml` 中改为：

```yaml
VITE_BASE_PATH: /仓库名/
```

## 可选：使用 username.github.io 仓库（免仓库名路径）

若创建仓库名为 **`你的用户名.github.io`**，则：

- 访问地址：`https://你的用户名.github.io/`（无仓库名后缀）
- `VITE_BASE_PATH` 保持 `/` 即可

Custom domain 绑定方式相同。
