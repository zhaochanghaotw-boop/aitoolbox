# 产品信息搜集模板

> 将本文件复制到**每个产品项目文件夹**中，在 Cursor 里打开该文件夹，用对话形式让 AI 帮你填写下方「待填写区」。
>
> 填写完成后，把内容汇总到导航站根目录的 `public/products.json`。

---

## 一、需要填写的字段（共 9 项）

| 字段 | 是否必填 | 说明 | 示例 |
|------|----------|------|------|
| `id` | ✅ 必填 | 产品唯一标识，英文小写 + 连字符，**全局不可重复** | `pdf-to-html` |
| `name` | ✅ 必填 | 产品展示名称，建议 4～12 字，中文 | `PDF 转 HTML` |
| `desc` | ✅ 必填 | 一句话简介，说明产品做什么、解决什么问题，建议 30～80 字 | `将 PDF 文档与图片转换为响应式 HTML 页面，精准还原排版布局。` |
| `version` | ✅ 必填 | 当前版本号，语义化版本 | `1.0.0` |
| `updated` | ✅ 必填 | 最近更新日期，格式 `YYYY-MM-DD` | `2026-06-29` |
| `tags` | ✅ 必填 | 标签数组，2～4 个，用于搜索和分类展示 | `["PDF", "HTML", "转换"]` |
| `demoUrl` | ✅ 必填 | **在线体验**链接，用户点击「在线体验」跳转的 H5/HTML 地址 | `https://oss.xxx.com/pdf-html/index.html` |
| `downloadUrl` | ✅ 必填 | **下载**链接，ZIP 包在 OSS 上的直链地址 | `https://oss.xxx.com/pdf-html.zip` |
| `cover` | ✅ 必填 | 封面图路径，放在导航站 `public/covers/` 下 | `/covers/pdf.svg` 或 `/covers/pdf.png` |

---

## 二、字段填写规范

### `id`（唯一标识）
- 只用英文字母、数字、连字符 `-`
- 不要用空格、中文、下划线
- 同一产品改名时 **id 不要改**，避免链接失效

### `name`（产品名称）
- 面向用户展示，简洁好记
- 可带「AI」前缀，但不要过长

### `desc`（产品描述）
- 写清楚：**输入什么 → 得到什么 → 核心价值**
- 不要写技术栈细节，用户看不懂的略去
- 一条描述即可，不要分段

### `version`（版本号）
- 格式：`主版本.次版本.修订号`，如 `1.2.3`
- 每次发版记得同步更新 `updated`

### `tags`（标签）
- 从以下类型里选 2～4 个：功能类型（PPT / PDF / VOC）、技术（AI / HTML）、场景（营销 / 效率 / 设计 / 分析）
- 标签会出现在卡片上，也会被搜索框匹配

### `demoUrl`（在线体验）
- 必须是**可直接在浏览器打开**的地址
- 通常是 OSS 上的 `index.html` 或部署后的 H5 链接
- 确保外链可公开访问（无登录墙）

### `downloadUrl`（下载地址）
- ZIP 压缩包的**直链**，点击即可下载
- 包内建议包含完整可运行文件（HTML + 资源）

### `cover`（封面图）
- 推荐尺寸：**800 × 480** 或 **5:3** 比例
- 格式：`.png` / `.jpg` / `.svg` 均可
- 文件放到导航站项目的 `public/covers/` 目录
- 路径写法：`/covers/你的产品id.png`

## 三、在 Cursor 里怎么搜集（复制下面这段话发给 AI）

```
请阅读当前文件夹里的代码和文档，帮我填写「产品信息搜集模板.md」底部的待填写区。

要求：
1. id：根据项目文件夹名或功能，给一个英文小写连字符格式的唯一 id
2. name：给一个有吸引力的中文产品名
3. desc：30～80 字，说清楚产品做什么、解决什么问题
4. version：从 package.json 或现有文档里找，没有就写 1.0.0
5. updated：今天日期，格式 YYYY-MM-DD
6. tags：根据功能给 2～4 个中文或英文标签
7. demoUrl：如果项目里有部署地址或 index.html 路径，帮我推断；没有就留空并标注「待补充」
8. downloadUrl：如果已知 OSS 地址就填，没有就留空并标注「待补充」
9. cover：建议文件名，如 /covers/xxx.png

如果某些信息在代码里找不到，请明确列出「还需要人工补充」的项。
```

## 六、汇总到导航站

所有产品信息搜集完成后，合并写入导航站根目录：

**文件路径：** `public/products.json`

**JSON 格式：**

```json
[
  {
    "id": "pdf-to-html",
    "name": "PDF 转 HTML",
    "desc": "将 PDF 文档与图片转换为响应式 HTML 页面，精准还原排版布局。",
    "version": "2.1.3",
    "updated": "2026-06-29",
    "tags": ["PDF", "HTML", "转换"],
    "demoUrl": "https://oss.xxx.com/pdf-html/index.html",
    "downloadUrl": "https://oss.xxx.com/pdf-html.zip",
    "cover": "/covers/pdf-to-html.png"
  }
]
```

> ⚠️ 注意：
> - `products.json` 是一个 **JSON 数组**，每个产品是一个对象
> - 对象之间用英文逗号 `,` 分隔，最后一个对象后面**不要**加逗号
> - 封面图文件需同步放到 `public/covers/` 目录
> - 改完后本地运行 `npm run dev` 预览，确认卡片展示正常
